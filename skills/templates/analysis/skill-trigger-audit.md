# Skill 트리거 감사 템플릿

## 사용 시기

- 여러 Skill 간에 트리거 문구가 겹치는지 확인해야 할 때
- 새 Skill 추가 시 기존 트리거와의 충돌을 사전 점검할 때
- 전체 Skill 포트폴리오의 트리거 체계를 정리해야 할 때
- 사용자가 의도한 Skill이 아닌 다른 Skill이 호출되는 문제를 해결할 때

## 프롬프트

당신은 Claude Code Skill 트리거 체계 감사 전문가입니다.
아래 Skill 목록의 트리거 문구를 전수 검사하고 충돌/중복을 식별하세요.

### 감사 규칙

다음 순서로 감사하세요:

1. **트리거 매트릭스 생성**
   - 모든 Skill의 모든 트리거를 하나의 매트릭스로 정리
   - 슬래시 명령어와 자연어 트리거를 분리

2. **충돌 유형 식별**
   - **완전 중복**: 동일한 트리거 문구가 2개 이상의 Skill에 존재
   - **부분 중복**: 한 트리거가 다른 트리거의 부분 문자열 (예: "커밋" vs "커밋 메시지 작성")
   - **의미 중복**: 다른 표현이지만 같은 의도 (예: "테스트 실행" vs "테스트 돌려줘")
   - **모호성**: 문맥에 따라 여러 Skill에 해당할 수 있는 트리거

3. **심각도 분류**
   - Critical: 완전 중복으로 항상 잘못된 Skill 호출 가능
   - Warning: 부분/의미 중복으로 간헐적 오호출 가능
   - Info: 잠재적 모호성이 있으나 현재 문제없음

4. **해결 방안 제시**
   - 각 충돌에 대해 구체적인 해결 방안 제시

### 출력 형식

```markdown
## Skill 트리거 감사 보고서

### 감사 개요

| 항목 | 내용 |
|------|------|
| 감사 일자 | {{AUDIT_DATE}} |
| 감사 대상 Skill 수 | {{SKILL_COUNT}}개 |
| 총 트리거 수 | {{TRIGGER_COUNT}}개 |
| 발견된 충돌 수 | {{CONFLICT_COUNT}}건 |

### 트리거 매트릭스

#### 슬래시 명령어

| Skill | 명령어 |
|-------|--------|
| {{SKILL_1}} | `/{{CMD_1}}` |
| {{SKILL_2}} | `/{{CMD_2}}` |

#### 자연어 트리거

| Skill | 트리거 문구 |
|-------|------------|
| {{SKILL_1}} | "{{NL_TRIGGER_1_1}}", "{{NL_TRIGGER_1_2}}" |
| {{SKILL_2}} | "{{NL_TRIGGER_2_1}}", "{{NL_TRIGGER_2_2}}" |

### 충돌 분석

#### Critical 충돌

| ID | Skill A | Skill B | 충돌 트리거 | 유형 | 해결 방안 |
|----|---------|---------|------------|------|-----------|
| C-1 | {{SKILL_A}} | {{SKILL_B}} | "{{CONFLICT_TRIGGER}}" | {{CONFLICT_TYPE}} | {{RESOLUTION}} |

#### Warning 충돌

| ID | Skill A | Skill B | 충돌 트리거 | 유형 | 해결 방안 |
|----|---------|---------|------------|------|-----------|
| W-1 | {{SKILL_A}} | {{SKILL_B}} | "{{CONFLICT_TRIGGER}}" | {{CONFLICT_TYPE}} | {{RESOLUTION}} |

#### Info 알림

| ID | Skill | 트리거 | 잠재적 모호성 | 권고 |
|----|-------|--------|-------------|------|
| I-1 | {{SKILL_A}} | "{{TRIGGER}}" | {{AMBIGUITY}} | {{ADVICE}} |
```
