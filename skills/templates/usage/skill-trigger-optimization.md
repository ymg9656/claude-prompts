# Skill 트리거 문구 최적화 템플릿

## 사용 시기

- Skill이 의도한 상황에서 호출되지 않을 때
- 트리거 문구가 너무 일반적이거나 너무 구체적일 때
- 사용자의 자연어 표현 다양성을 반영하고 싶을 때
- 다른 Skill과의 트리거 충돌을 해결해야 할 때

## 프롬프트

당신은 Claude Code Skill 트리거 최적화 전문가입니다.
아래 Skill의 트리거 문구를 분석하고 개선안을 제시하세요.

### 입력 정보

- **Skill 이름**: {{SKILL_NAME}}
- **현재 트리거 문구**: {{CURRENT_TRIGGERS}}
<!-- 현재 설정된 트리거 문구 전체를 나열하세요 -->
- **Skill 핵심 기능**: {{SKILL_CORE_FUNCTION}}
- **의도한 호출 상황**: {{INTENDED_SCENARIOS}}
<!-- 이 Skill이 호출되어야 하는 구체적 상황 목록 -->
- **오호출 사례**: {{?FALSE_POSITIVE_CASES}}
<!-- 의도하지 않게 호출된 사례 -->
- **미호출 사례**: {{?FALSE_NEGATIVE_CASES}}
<!-- 호출되어야 했지만 안 된 사례 -->
- **관련 Skill 목록**: {{?RELATED_SKILLS}}
<!-- 비슷한 영역의 다른 Skill 이름과 트리거 -->

### 분석 및 최적화 규칙

다음 순서로 분석하세요:

1. **현재 트리거 분석**
   - 각 트리거의 특이성(specificity) 점수 (1-5)
   - 사용자 자연어 커버리지 평가
   - 다른 Skill과의 충돌 가능성

2. **개선안 설계**
   - 슬래시 명령어: 1~2개 (핵심 동사 기반)
   - 자연어 트리거: 5~10개 (다양한 표현 패턴)
   - 부정 트리거(이 Skill이 아닌 경우): 2~3개

3. **검증 시나리오**
   - 각 개선 트리거에 대해 호출 시나리오 작성
   - 엣지 케이스 식별

### 출력 형식

```markdown
## 현재 트리거 분석

| 트리거 | 특이성 | 문제점 |
|--------|--------|--------|
| {{TRIGGER_1}} | {{SPECIFICITY_SCORE}}/5 | {{ISSUE}} |

## 개선된 트리거

### 슬래시 명령어
- `/{{PRIMARY_COMMAND}}`
- `/{{?ALIAS_COMMAND}}`

### 자연어 트리거 (호출됨)
- "{{NATURAL_TRIGGER_1}}"
- "{{NATURAL_TRIGGER_2}}"
- "{{NATURAL_TRIGGER_3}}"
- "{{NATURAL_TRIGGER_4}}"
- "{{NATURAL_TRIGGER_5}}"

### 부정 트리거 (호출되지 않음)
- "{{NEGATIVE_TRIGGER_1}}" → 대신 {{ALTERNATIVE_SKILL}} 사용
- "{{NEGATIVE_TRIGGER_2}}" → 대신 {{ALTERNATIVE_SKILL}} 사용

## 검증 시나리오

| 시나리오 | 사용자 입력 | 기대 결과 |
|----------|------------|-----------|
| 정상 호출 | "{{SCENARIO_INPUT_1}}" | {{SKILL_NAME}} 호출됨 |
| 정상 호출 | "{{SCENARIO_INPUT_2}}" | {{SKILL_NAME}} 호출됨 |
| 미호출 | "{{SCENARIO_INPUT_3}}" | 다른 Skill 또는 일반 대화 |
```

## 품질 기준

- [ ] 슬래시 명령어가 1~2개로 간결한가?
- [ ] 자연어 트리거가 5개 이상이며 다양한 표현을 커버하는가?
- [ ] 부정 트리거가 명시되어 경계가 명확한가?
- [ ] 다른 Skill과의 충돌이 해소되었는가?
- [ ] 검증 시나리오가 정상/비정상 케이스를 모두 포함하는가?
- [ ] 트리거가 사용자의 자연스러운 한국어/영어 표현을 반영하는가?
