# Agent 작업 위임

## 사용 시기

- Agent에게 구체적인 작업을 전달하고 높은 품질의 결과를 얻고 싶을 때
- 작업의 범위, 기대치, 제약 조건을 명확하게 정의해야 할 때
- 반복적으로 동일한 유형의 작업을 Agent에게 위임하는 표준 형식이 필요할 때

## 프롬프트

다음 형식에 따라 Agent에게 위임할 작업을 정의하세요.

---

### 작업 위임 명세서

```
## 작업 개요

### 대상 Agent
{{AGENT_NAME}}

### 작업 유형
{{TASK_TYPE}}
<!-- TASK_TYPE: 코드 리뷰 / 버그 수정 / 기능 구현 / 리팩토링 / 문서 작성 / 테스트 작성 / 아키텍처 설계 등 -->

### 작업 요약
{{TASK_SUMMARY}}
<!-- 1~2문장으로 작업의 핵심을 설명 -->

---

## 상세 지시사항

### 목표
{{TASK_OBJECTIVE}}
<!-- 작업 완료 시 달성해야 하는 구체적 목표 -->

### 대상 범위
- 파일/경로: {{TARGET_FILES}}
- 관련 모듈: {{?RELATED_MODULES}}
- 제외 대상: {{?EXCLUDED_FILES}}

### 작업 단계
1. {{STEP_1}}
2. {{STEP_2}}
3. {{STEP_3}}
{{?STEP_4}}
{{?STEP_5}}

### 입력 정보
{{INPUT_DATA}}
<!-- Agent에게 제공하는 참고 정보: 에러 로그, 요구사항 문서, 관련 코드 등 -->

---

## 기대 출력

### 출력 형식
{{OUTPUT_FORMAT}}
<!-- 예: 마크다운 리포트, 코드 패치, PR 코멘트, 설계 문서 등 -->

### 출력 예시
{{?OUTPUT_EXAMPLE}}
<!-- 기대하는 결과물의 형태를 간략히 보여줌 -->

### 성공 기준
- {{SUCCESS_CRITERION_1}}
- {{SUCCESS_CRITERION_2}}
- {{?SUCCESS_CRITERION_3}}

---

## 제약 조건

### 기술적 제약
- {{TECHNICAL_CONSTRAINT_1}}
- {{?TECHNICAL_CONSTRAINT_2}}

### 행동 제약
- {{BEHAVIORAL_CONSTRAINT_1}}
- {{?BEHAVIORAL_CONSTRAINT_2}}
<!-- 예: "기존 테스트를 깨뜨리지 않을 것", "public API를 변경하지 않을 것" -->

### 우선순위
{{PRIORITY}}
<!-- 긴급(즉시) / 높음(오늘 내) / 보통(이번 주) / 낮음(시간 여유) -->

---

## 컨텍스트 보충

### 배경 정보
{{?BACKGROUND_CONTEXT}}
<!-- 작업이 필요한 이유, 비즈니스 배경 등 -->

### 이전 시도
{{?PREVIOUS_ATTEMPTS}}
<!-- 이전에 시도했으나 실패한 접근 방식 (있다면) -->

### 참고 자료
{{?REFERENCE_MATERIALS}}
<!-- 관련 문서, 이슈 링크, 디자인 문서 등 -->
```

---

### 위임 체크리스트

작업을 전달하기 전에 다음을 확인하세요:

- [ ] 작업 목표가 한 문장으로 요약 가능한가?
- [ ] 대상 파일/범위가 구체적으로 지정되었는가?
- [ ] Agent가 모르는 프로젝트 특이사항이 컨텍스트에 포함되었는가?
- [ ] 성공 기준이 검증 가능한가?
- [ ] 제약 조건이 명시적인가? (암묵적 기대 금지)

## 품질 기준

- [ ] 작업 범위가 Agent 하나가 처리하기에 적절한 크기인가?
- [ ] 목표와 성공 기준이 일치하는가?
- [ ] 입력 정보가 Agent 작업에 충분한가? (부족한 정보 없음)
- [ ] 제약 조건이 현실적인가? (상충되는 제약 없음)
- [ ] 출력 형식이 후속 작업에 활용 가능한가?
- [ ] 컨텍스트 윈도우를 효율적으로 사용하는가? (불필요한 정보 배제)
