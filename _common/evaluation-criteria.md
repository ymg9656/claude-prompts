# 공통 평가 기준

프롬프트 결과물의 품질을 검증하는 공통 체크리스트입니다.

---

## 1. 명확성 (Clarity)

- [ ] 모호한 표현 없이 의도가 명확한가?
- [ ] 전문 용어가 적절히 사용되었는가?
- [ ] 한 문장이 하나의 의미만 전달하는가?

## 2. 완전성 (Completeness)

- [ ] 필수 플레이스홀더가 모두 채워졌는가?
- [ ] 입력/출력 형식이 명시되었는가?
- [ ] 예외 상황에 대한 처리가 있는가?

## 3. 구조 (Structure)

- [ ] 표준 3-파트 구조(사용 시기 / 프롬프트 / 품질 기준)를 따르는가?
- [ ] 논리적 순서로 단계가 배치되었는가?
- [ ] 불필요한 중복이 없는가?

## 4. 실용성 (Practicality)

- [ ] 실제 워크플로우에서 바로 사용할 수 있는가?
- [ ] 컨텍스트 윈도우를 효율적으로 사용하는가?
- [ ] 필요한 도구/권한이 명시되었는가?

## 5. 재현성 (Reproducibility)

- [ ] 동일 입력에 대해 일관된 결과를 생성하는가?
- [ ] 프롬프트를 다른 프로젝트에 재사용할 수 있는가?
- [ ] 버전/날짜에 의존적인 정보가 없는가?

---

## 카테고리별 추가 기준

### Agent
- [ ] 역할과 권한 범위가 명확히 정의되었는가?
- [ ] 다른 Agent와의 경계가 명확한가?
- [ ] 도구 사용 지침이 구체적인가?

### Skill
- [ ] 트리거 문구가 자연스럽고 구분 가능한가?
- [ ] 워크플로우 단계가 독립적으로 실행 가능한가?
- [ ] 에러 처리 시나리오가 포함되었는가?

### Team
- [ ] 멤버 역할이 겹치지 않는가?
- [ ] 커뮤니케이션 흐름이 명확한가?
- [ ] 병목 지점이 식별되었는가?

### Docs
- [ ] 대상 독자가 명확한가?
- [ ] 목차와 내용이 일치하는가?
- [ ] 코드 예시가 실행 가능한가?

### Conversation
- [ ] 페르소나가 일관되게 유지되는가?
- [ ] 대화 흐름이 자연스러운가?
- [ ] 출력 형식이 사용 목적에 맞는가?
