# conv-response-analysis -- 응답 품질 분석

## 사용 시기

- Claude의 응답이 기대와 다를 때 원인을 분석하고 싶을 때
- 여러 응답 간 품질 차이를 비교 분석해야 할 때
- 응답의 정확성, 완전성, 유용성을 체계적으로 검증해야 할 때
- 할루시네이션, 편향, 누락 등의 문제를 식별하고 싶을 때

다음 분석 프레임워크에 따라 응답을 분석하세요:

### 1. 정확성 분석 (Accuracy)
- **사실 정확성**: 기술적/사실적 오류가 있는가?
- **코드 정확성**: 코드가 포함된 경우, 실행 가능하고 올바른가?
- **할루시네이션 탐지**: 존재하지 않는 API, 함수, 개념을 언급하는가?
- 발견된 오류를 구체적으로 나열하세요.

### 2. 완전성 분석 (Completeness)
- **요청 충족도**: 프롬프트의 모든 요구사항에 답했는가?
- **누락 항목**: 빠뜨린 요소가 있는가?
- **깊이 적절성**: 충분한 깊이로 다루었는가, 피상적인가?

### 3. 관련성 분석 (Relevance)
- **초점 유지**: 주제에서 벗어난 내용이 있는가?
- **잡음 비율**: 불필요한 부연 설명이나 반복이 있는가?
- **우선순위**: 중요한 정보가 먼저 제시되었는가?

### 4. 사용성 분석 (Usability)
- **즉시 활용 가능성**: 응답을 바로 사용할 수 있는가?
- **가독성**: 구조와 포맷이 읽기 쉬운가?
- **실행 가능성**: 제안이 구체적이고 실행 가능한가?

### 5. 일관성 분석 (Consistency)
- **내적 일관성**: 응답 내에서 모순되는 내용이 있는가?
- **톤 일관성**: 전체적으로 톤과 스타일이 균일한가?
- **용어 일관성**: 같은 개념에 대해 일관된 용어를 사용하는가?
