# conv-prompt-evaluation -- 프롬프트 효과성 평가

## 사용 시기

- 작성한 프롬프트가 의도한 결과를 생성하는지 검증하고 싶을 때
- 여러 프롬프트 버전 중 가장 효과적인 것을 선택해야 할 때
- 프롬프트의 약점을 체계적으로 분석하여 개선 방향을 도출할 때
- 프롬프트 라이브러리에 등록하기 전 품질 게이트를 통과시킬 때

## 프롬프트

```markdown
당신은 프롬프트 엔지니어링 전문가입니다.
다음 프롬프트의 효과성을 체계적으로 평가하세요.

### 평가 대상 프롬프트
```
{{TARGET_PROMPT}}
```

### 프롬프트의 의도된 목적
{{INTENDED_PURPOSE}}
<!-- 예: "주니어 개발자에게 코드 리뷰 피드백을 생성하는 것" -->

### 대상 모델
{{TARGET_MODEL}}
<!-- 예: "Claude 3.5 Sonnet", "Claude Opus", "GPT-4" -->

### 실제 응답 샘플 (선택)
{{?RESPONSE_SAMPLES}}
<!-- 이 프롬프트로 생성된 실제 응답 1-3개. 있으면 평가 정확도가 높아집니다 -->

---

다음 5개 차원에서 평가하세요. 각 차원은 1-5점으로 채점합니다.

### 평가 차원

**1. 명확성 (Clarity) - __ /5**
- 지시사항이 모호함 없이 해석되는가?
- 역할, 작업, 출력 형식이 명확한가?
- 하나의 지시가 하나의 의미만 전달하는가?

**2. 구체성 (Specificity) - __ /5**
- 추상적 지시 대신 구체적 기준이 있는가?
- 예시나 템플릿이 기대 수준을 보여주는가?
- 모호한 형용사 ("좋은", "적절한") 대신 측정 가능한 기준이 있는가?

**3. 구조 (Structure) - __ /5**
- 논리적 순서로 정보가 배치되었는가?
- 섹션 구분이 명확한가?
- 불필요한 중복이 없는가?

**4. 완전성 (Completeness) - __ /5**
- 작업 수행에 필요한 모든 정보가 포함되었는가?
- 엣지 케이스나 예외 처리 지침이 있는가?
- 출력 형식이 명시되었는가?

**5. 효율성 (Efficiency) - __ /5**
- 토큰 수 대비 정보 밀도가 높은가?
- 불필요한 수식어나 반복이 없는가?
- 컨텍스트 윈도우를 낭비하지 않는가?

### 출력 형식

다음 형식으로 평가 결과를 작성하세요:

```markdown
## 프롬프트 평가 리포트

### 총점: __ / 25

### 차원별 점수
| 차원 | 점수 | 주요 소견 |
|------|------|-----------|
| 명확성 | /5 | |
| 구체성 | /5 | |
| 구조 | /5 | |
| 완전성 | /5 | |
| 효율성 | /5 | |

### 강점 (유지할 부분)
1.
2.

### 약점 (개선할 부분)
1.
2.

### 구체적 개선 제안
1. [약점 1에 대한 구체적 수정안]
2. [약점 2에 대한 구체적 수정안]

### 개선된 프롬프트 (선택)
[점수가 15점 미만일 경우 전체 재작성 버전 제공]
```
```

## 품질 기준

- [ ] 5개 평가 차원 모두에 대해 구체적 근거와 함께 점수가 매겨졌는가?
- [ ] 강점과 약점이 균형 있게 식별되었는가? (편향 없는 평가)
- [ ] 개선 제안이 추상적 조언이 아닌 구체적 수정안을 포함하는가?
- [ ] 평가가 프롬프트의 의도된 목적과 대상 모델을 고려하고 있는가?
- [ ] 실제 응답 샘플이 제공된 경우, 응답 분석이 평가에 반영되었는가?
- [ ] 점수 기준이 일관되고 재현 가능한가? (다시 평가해도 비슷한 결과)
