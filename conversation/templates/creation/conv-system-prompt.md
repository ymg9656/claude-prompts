# conv-system-prompt -- 대화용 시스템 프롬프트 설계

## 사용 시기

- Claude와의 대화에 사용할 시스템 프롬프트를 새로 설계할 때
- 특정 도메인이나 업무에 특화된 대화 환경을 구축할 때
- 기존 시스템 프롬프트를 구조화된 형식으로 재작성할 때
- API 호출 시 `system` 파라미터에 전달할 프롬프트가 필요할 때

## 프롬프트

```markdown
당신은 시스템 프롬프트 설계 전문가입니다.
다음 요구사항에 맞는 Claude 대화용 시스템 프롬프트를 설계하세요.

### 대화 목적
<!-- 이 대화가 달성하려는 핵심 목표를 기술합니다 -->
{{CONVERSATION_PURPOSE}}

### 대상 사용자
<!-- 이 시스템 프롬프트를 통해 대화할 사용자의 특성 -->
- 역할: {{USER_ROLE}}
- 기술 수준: {{USER_EXPERTISE_LEVEL}}
<!-- 예: 초급/중급/고급, 또는 "Python 2년 경험의 백엔드 개발자" -->
{{?USER_ADDITIONAL_TRAITS}}

### 도메인 컨텍스트
- 분야: {{DOMAIN}}
<!-- 예: 웹 개발, 데이터 분석, 법률 자문 등 -->
- 핵심 주제: {{KEY_TOPICS}}
<!-- 예: REST API 설계, Spring Security, OAuth 2.0 -->
- 사용 기술/도구: {{?TECH_TOOLS}}
<!-- 예: Java 17, Spring Boot 3.x, PostgreSQL -->

### 응답 스타일 요구사항
- 톤: {{TONE}}
<!-- 예: 전문적이고 간결한 / 친근하고 설명적인 / 학술적인 -->
- 응답 길이: {{RESPONSE_LENGTH}}
<!-- 예: 간결 (1-3문장) / 중간 (1-2단락) / 상세 (필요한 만큼) -->
- 언어: {{RESPONSE_LANGUAGE}}
<!-- 예: 한국어 / 영어 / 한국어 (기술 용어는 영어) -->
{{?STYLE_EXAMPLES}}
<!-- 원하는 스타일의 예시 응답이 있다면 여기에 포함 -->

### 행동 규칙
<!-- Claude가 반드시 따라야 할 규칙 목록 -->
1. {{RULE_1}}
2. {{RULE_2}}
3. {{RULE_3}}
{{?ADDITIONAL_RULES}}

### 제한 사항
<!-- Claude가 하지 말아야 할 것들 -->
- {{RESTRICTION_1}}
- {{RESTRICTION_2}}
{{?ADDITIONAL_RESTRICTIONS}}

### 출력 형식 (선택)
{{?OUTPUT_FORMAT}}
<!-- 특정 출력 형식이 필요한 경우 (예: JSON, Markdown 표, 코드 블록) -->

---

위 요구사항을 바탕으로 다음 구조의 시스템 프롬프트를 생성하세요:

1. **역할 선언** (1-2문장): "당신은 ~입니다" 형태
2. **핵심 지침** (3-5개 항목): 응답 시 항상 따를 규칙
3. **도메인 지식 범위**: 다룰 주제와 다루지 않을 주제
4. **응답 형식 가이드**: 기본 응답 구조
5. **예외 처리**: 모르거나 범위 밖인 질문에 대한 대응 방법

생성된 프롬프트는 즉시 API의 system 파라미터에 사용할 수 있는 형태여야 합니다.
```

## 품질 기준

- [ ] 역할 선언이 구체적이고 명확한가? ("전문가" 같은 모호한 표현 대신 구체적 역할)
- [ ] 핵심 지침이 3~5개로 한정되어 있는가? (너무 많으면 우선순위 불명확)
- [ ] 도메인 범위가 명확하게 정의되었는가? (다룰 것과 다루지 않을 것의 경계)
- [ ] 응답 톤과 길이가 대상 사용자에게 적합한가?
- [ ] 제한 사항이 부정형으로 명확히 기술되었는가? ("~하지 마세요")
- [ ] 생성된 프롬프트가 과도한 컨텍스트 없이 간결한가? (토큰 효율성)
- [ ] 예외 상황에 대한 안내가 포함되었는가? (모르는 질문, 범위 밖 요청)
