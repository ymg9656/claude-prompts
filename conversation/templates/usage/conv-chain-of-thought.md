# conv-chain-of-thought -- Chain-of-Thought 추론 유도

## 사용 시기

- 복잡한 분석이나 의사결정이 필요한 질문을 할 때
- Claude의 답변에 논리적 근거와 추론 과정을 포함시키고 싶을 때
- 단순 답변보다 사고 과정이 중요한 문제 (디버깅, 설계 판단, 트레이드오프 분석)
- 다단계 추론이 필요한 작업에서 중간 단계의 검증이 필요할 때
- 오류 가능성이 높은 작업에서 자기 검증을 유도하고 싶을 때

## 프롬프트

```markdown
당신은 {{ROLE}} 분야의 전문가입니다.
다음 문제에 대해 단계적으로 사고하며 답변하세요.

### 문제
{{PROBLEM_STATEMENT}}

### 배경 정보
{{?CONTEXT_INFORMATION}}
<!-- 문제 해결에 필요한 추가 맥락, 코드, 데이터 등 -->

### 제약 조건
{{?CONSTRAINTS}}
<!-- 예: 성능 요구사항, 기술 스택 제한, 시간/비용 제약 -->

### 추론 방식

다음 단계를 따라 사고하세요:

**1단계: 문제 분해**
- 핵심 질문을 식별하세요
- 하위 문제로 분해하세요
- 각 하위 문제의 의존 관계를 파악하세요

**2단계: {{REASONING_STEP_2}}**
<!-- 문제 유형에 따라 커스터마이즈 -->
<!-- 예 (디버깅): "가설 수립 - 가능한 원인을 나열하고 각각의 가능성을 평가하세요" -->
<!-- 예 (설계): "선택지 도출 - 가능한 접근 방식을 나열하세요" -->
<!-- 예 (분석): "데이터 검토 - 주어진 정보에서 패턴을 찾으세요" -->

**3단계: {{REASONING_STEP_3}}**
<!-- 예 (디버깅): "검증 - 가장 유력한 가설을 검증하세요" -->
<!-- 예 (설계): "비교 평가 - 각 선택지의 장단점을 비교하세요" -->
<!-- 예 (분석): "해석 - 발견한 패턴의 의미를 해석하세요" -->

**4단계: 결론 및 자기 검증**
- 최종 답변을 제시하세요
- 자신의 추론에서 약한 부분이 있다면 명시하세요
- 추가 검증이 필요한 가정이 있다면 밝히세요

### 출력 형식

각 단계의 사고 과정을 명시적으로 보여주세요:

```
## 1단계: 문제 분해
[사고 과정]

## 2단계: [단계명]
[사고 과정]

## 3단계: [단계명]
[사고 과정]

## 4단계: 결론
[최종 답변]

### 신뢰도 자가 평가
- 확신도: [높음/중간/낮음]
- 불확실한 부분: [있다면 명시]
- 추가 확인 필요 사항: [있다면 명시]
```

{{?ADDITIONAL_INSTRUCTIONS}}
<!-- 예: "코드로 설명할 수 있는 부분은 코드 예시를 포함하세요" -->
```

## 품질 기준

- [ ] 각 추론 단계가 명시적으로 구분되어 있는가?
- [ ] 문제 분해 단계에서 핵심 하위 문제가 올바르게 식별되었는가?
- [ ] 중간 단계의 추론이 논리적으로 연결되어 있는가? (비약 없음)
- [ ] 결론이 추론 과정에서 자연스럽게 도출되는가?
- [ ] 자기 검증 단계에서 불확실한 부분이 솔직하게 명시되었는가?
- [ ] 추론 과정이 검증 가능한 수준으로 구체적인가? (다른 사람이 따라갈 수 있는가?)
- [ ] 불필요하게 장황하지 않으면서 충분한 깊이를 갖추고 있는가?
