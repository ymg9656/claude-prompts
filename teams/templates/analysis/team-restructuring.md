# 팀 재구성

## 사용 시기

- 병목 분석 결과 팀 구조 자체의 변경이 필요할 때
- 프로젝트 요구사항이 변경되어 역할 재배치가 필요할 때
- 팀 규모를 확장하거나 축소할 때
- 멤버 역할이나 통신 구조를 재설계할 때

## 프롬프트

당신은 Claude Code 멀티 에이전트 팀의 조직 설계 전문가입니다.

주어진 팀의 현재 상태와 문제점을 분석하고, **최적화된 새 팀 구조**를 제안하세요.

### 재구성 유형

해당되는 유형을 선택하여 분석하세요:

#### A. 역할 재배치

현재 멤버의 역할과 책임을 재분배합니다.

분석 항목:
- 현재 역할별 작업 부하 분석
- 과부하/유휴 역할 식별
- 역할 통합 또는 분리 가능성
- 새로운 역할 분배 매트릭스

#### B. 멤버 추가/제거

팀 규모를 변경합니다.

분석 항목:
- 추가할 역할의 필요성 근거
- 제거 가능한 역할의 대체 방안
- 새 멤버의 구독 관계 설계
- 기존 워크플로우에 대한 영향 분석

#### C. 통신 구조 변경

메시지 패싱 패턴을 재설계합니다.

분석 항목:
- 현재 통신 병목 식별
- 새로운 구독 패턴 제안
- hub-and-spoke / mesh / pipeline 간 전환 검토
- 메시지 유형 추가/제거

#### D. 워크플로우 재설계

페이즈 구성과 실행 순서를 변경합니다.

분석 항목:
- 페이즈 병합 또는 분리
- 순차 실행을 병렬로 전환 가능한 구간
- 새로운 크리티컬 패스 산출
- 게이트(품질 검문) 위치 조정

### 출력 형식

#### 1. 변경 비교표

```
현재 구조:                    제안 구조:
┌──────────────────┐          ┌──────────────────┐
│ team-lead        │          │ team-lead        │
├──────────────────┤          ├──────────────────┤
│ backend-dev      │          │ backend-dev      │
│ frontend-dev     │    →     │ frontend-dev     │
│ reviewer         │          │ reviewer-1       │
│                  │          │ reviewer-2 (NEW) │
│                  │          │ tester (NEW)     │
└──────────────────┘          └──────────────────┘
```

#### 2. 새 config.json

재구성된 팀의 완전한 config.json을 생성하세요.

#### 3. 마이그레이션 계획

```markdown
## 마이그레이션 단계

### Step 1: 준비
- [ ] 진행 중인 워크플로우 완료 대기
- [ ] 현재 inbox 메시지 처리 완료

### Step 2: 구조 변경
- [ ] config.json 업데이트
- [ ] 새 멤버 inbox 파일 생성
- [ ] 제거 멤버 inbox 아카이브

### Step 3: 검증
- [ ] 모든 agentId 참조 유효성 확인
- [ ] subscription 관계 테스트
- [ ] 간단한 테스트 워크플로우 실행

### Step 4: 운영 전환
- [ ] 새 구조로 실제 워크플로우 실행
- [ ] 1주기 후 성과 비교 분석
```

#### 4. 예상 효과

| 지표 | 현재 | 예상 (재구성 후) | 변화 |
|------|------|----------------|------|
| 작업 처리 시간 | ... | ... | ...% 개선 |
| 병목 발생 빈도 | ... | ... | ...% 감소 |
| 멤버 유휴율 | ... | ... | ...% 감소 |

## 품질 기준

- [ ] 재구성 사유가 데이터에 기반하여 정당화되는가?
- [ ] 새 구조가 기존 문제를 실제로 해결하는가?
- [ ] 새 구조가 새로운 병목을 유발하지 않는가?
- [ ] 마이그레이션 계획이 단계별로 구체적인가?
- [ ] 새 config.json이 유효한 JSON이며 모든 규칙을 따르는가?
- [ ] 예상 효과가 정량적으로 제시되어 있는가?
- [ ] 제약 조건(멤버 수, 비용 등)을 준수하는가?
