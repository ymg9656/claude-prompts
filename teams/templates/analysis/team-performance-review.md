### 분석 항목

다음 관점에서 성과를 분석하세요:

#### 1. 팀 전체 성과

- **작업 완료율**: 할당된 작업 중 완료된 비율
- **품질 통과율**: 첫 번째 제출에서 승인된 비율 (재작업 없이)
- **평균 작업 소요 시간**: 작업 할당부터 완료까지의 시간
- **병목 발생 빈도**: 블로커/지연이 발생한 횟수

#### 2. 멤버별 기여도

각 멤버에 대해 분석:

| 멤버 | 할당 작업 | 완료 작업 | 첫 통과율 | 평균 소요 시간 | 도움 요청 수 | 피드백 반영률 |
|------|----------|----------|----------|--------------|-------------|-------------|
| ... | ... | ... | ... | ... | ... | ... |

#### 3. 커뮤니케이션 효율성

- **총 메시지 수**: 유형별 메시지 발생량
- **응답 시간**: 메시지 발송 후 응답까지의 평균 시간
- **재작업 요청 빈도**: 피드백으로 인한 수정 작업 횟수
- **불필요 메시지 비율**: 실질적 정보가 없는 메시지 비율

#### 4. 워크플로우 효율성

- **페이즈별 소요 시간**: 예상 대비 실제 소요 시간
- **병렬화 활용률**: 병렬 실행 가능했던 작업 중 실제 병렬 실행된 비율
- **대기 시간 비율**: 전체 시간 중 멤버가 대기한 시간 비율

### 출력 형식

```markdown
# {{TEAM_NAME}} 성과 분석 리포트

## 요약
- 전체 평가: A / B / C / D
- 핵심 강점: ...
- 주요 개선점: ...

## 상세 분석

### 1. 팀 전체 성과
(위 항목들의 수치와 분석)

### 2. 멤버별 기여도
(테이블 + 개별 코멘트)

### 3. 커뮤니케이션 효율성
(수치 분석 + 패턴 식별)

### 4. 워크플로우 효율성
(병목 분석 + 최적화 제안)

## 개선 권고사항
1. 단기 (즉시 적용 가능): ...
2. 중기 (다음 사이클): ...
3. 장기 (구조적 변경): ...
```

## 품질 기준

- [ ] 정량적 수치(비율, 시간, 횟수)가 포함되어 있는가?
- [ ] 멤버별 분석이 공정하고 객관적 데이터에 기반하는가?
- [ ] 개선 권고사항이 구체적이고 실행 가능한가?
- [ ] 커뮤니케이션 패턴에 대한 분석이 포함되어 있는가?
- [ ] 워크플로우 병목이 정확히 식별되어 있는가?
- [ ] 분석 결과가 다음 팀 운영에 활용 가능한 형태인가?
