# 팀 작업 분배

## 사용 시기

- 새로운 작업(기능, 버그 수정, 리팩토링 등)을 팀 멤버에게 분배할 때
- 대규모 작업을 서브태스크로 분해하고 적절한 멤버에게 할당할 때
- 작업 우선순위와 의존 관계를 고려한 할당 계획을 수립할 때

### 분배 전략

다음 절차로 작업을 분배하세요:

**1단계: 작업 분해**
- 주어진 작업을 독립적으로 수행 가능한 서브태스크로 분해
- 각 서브태스크의 예상 복잡도 표시 (S / M / L / XL)
- 서브태스크 간 의존 관계 식별

**2단계: 멤버 매칭**
- 각 서브태스크에 가장 적합한 멤버를 매칭
- 매칭 기준: 전문 분야 일치도 > 현재 가용성 > 작업 균형
- 한 멤버에게 과도한 작업이 집중되지 않도록 균형 유지

**3단계: 실행 순서 결정**
- 의존 관계를 고려한 실행 순서 도출
- 병렬 실행 가능한 태스크 그룹 식별
- 크리티컬 패스 식별

**4단계: 할당 메시지 생성**
각 멤버에게 보낼 메시지를 다음 형식으로 생성:

```json
{
  "from": "team-lead@{{TEAM_NAME}}",
  "to": "<member-agentId>",
  "type": "task-assignment",
  "payload": {
    "taskId": "TASK-XXX",
    "title": "서브태스크 제목",
    "description": "상세 설명",
    "acceptanceCriteria": [
      "완료 기준 1",
      "완료 기준 2"
    ],
    "dependencies": ["선행 태스크 ID"],
    "priority": "high",
    "estimatedComplexity": "M"
  }
}
```

### 출력 형식

#### 작업 분해 결과

| ID | 서브태스크 | 복잡도 | 담당 멤버 | 의존 관계 | 실행 순서 |
|----|----------|--------|----------|----------|----------|
| TASK-001 | ... | M | ... | 없음 | 1 |
| TASK-002 | ... | L | ... | TASK-001 | 2 |

#### 실행 타임라인

```
Phase 1 (병렬): TASK-001, TASK-003
Phase 2 (TASK-001 완료 후): TASK-002
Phase 3 (전체 완료 후): TASK-004 (통합 테스트)
```
